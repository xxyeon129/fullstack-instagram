# 1. 사용자 시나리오

## 1-1. 회원가입
1. 이메일, 비밀번호, 사용자 이름으로 회원가입 요청

2. 이메일 / 사용자 이름 중복 여부 검증

3. 비밀번호 암호화 (bcrypt 해싱??)

<br />

## 1-2. 로그인
1. 사용자는 이메일 + 비밀번호로 로그인

2. Access Token(JWT, 15분), Refresh Token(7일)을 발급

3. 이후 모든 인증이 필요한 요청은 `Authorization: Bearer <token>` 헤더를 포함

4. Access Token 만료 시, Refresh Token으로 재발급 요청

<br />

## 1-3. 게시물 업로드
1. 이미지(최대 10장) + 캡션 텍스트 첨부하여 업로드 요청

2. 서버는 이미지를 S3에 업로드하고 URL 반환

3. 게시물(Post) 레코드 / 이미지(PostImage) 레코드를 DB에 저장

4. 저장 완료 후 게시물 상세 응답 반환

5. 썸네일 생성 등 비동기 처리 수행

<br />

## 1-4. 피드 조회
1. 사용자가 팔로우한 사람들의 홈 피드(게시글 리스트) 요청

2. 서버는 팔로우 목록을 조회하고, 해당 사용자들의 최신 게시물을 페이지네이션으로 반환

3. 각 게시물에는 이미지 목록, 좋아요/댓글 수, 북마크/좋아요 여부 상태가 포함됨

4. 피드 목록 Redis 캐시에서 우선 조회하고, 없을 경우 DB에서 조회 후 캐시에 저장

<br />

## 1-5. 좋아요 / 댓글 / 북마크
### 좋아요 기능
1. 게시물에 좋아요를 누르면 likes 레코드가 생성

2. 이미 좋아요 상태에서 다시 누를 경우 좋아요 삭제 처리 (레코드 삭제)

### 댓글 기능
1. 게시물에 댓글 작성 가능

2. 댓글에 대한 대댓글(1-depth) 작성 가능

3. 본인이 작성한 댓글만 삭제 가능

### 북마크 기능
1. 게시물을 북마크에 저장/해제 가능

2. 내 북마크 목록 페이지네이션으로 조회 가능

<br />

## 1-6. 팔로우 / 언팔로우
1. 다른 사용자 팔로우 가능

2. 자기 자신은 팔로우 불가

3. 이미 팔로우 중이면 언팔로우

4. 팔로워 / 팔로잉 목록 조회 가능
